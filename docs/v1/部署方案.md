# RAG-LAB 部署方案汇总

针对 `rag-lab` 这个轻量级的单进程 RAG MVP 项目，以下提供几种从简单到进阶的部署方案，你可以根据实际使用场景（个人测试、团队内部使用或对外展示）进行选择。

---

## 方案一：最简 Linux/云主机单机部署（适合个人使用）
这是最直接的方式，适合拥有一台云服务器（如阿里云、腾讯云、AWS EC2）的开发者。

**部署步骤**：
1. 在服务器上克隆代码。
2. 安装 Python 环境（推荐 3.10+）。
3. 激活虚拟环境并安装依赖：`pip install -e .` 或 `pip install -r requirements.txt`。
4. 配置 `.env` 文件填入 `OPENAI_API_KEY` 等参数。
5. **后台运行 Web UI**：
   使用 `nohup` 或 `tmux`/`screen` 保持后台运行：
   ```bash
   nohup streamlit run app.py --server.port 8501 --server.address 0.0.0.0 > streamlit.log 2>&1 &
   ```
6. 配置服务器的安全组/防火墙，放行 `8501` 端口。

---

## 方案二：Docker 容器化部署（推荐，适合团队环境）
容器化能免去环境冲突的烦恼，便于迁移和扩展，是工程实践的最佳选择。

**部署步骤**：
1. **编写 Dockerfile**：在项目根目录创建如下配置
   ```dockerfile
   FROM python:3.10-slim
   WORKDIR /app
   COPY . /app
   RUN pip install --no-cache-dir -e .
   EXPOSE 8501
   CMD ["streamlit", "run", "app.py", "--server.port", "8501", "--server.address", "0.0.0.0"]
   ```
2. **构建镜像**：
   ```bash
   docker build -t rag-lab:v1 .
   ```
3. **运行容器**（映射本地的 `data` 和 `chroma_db` 以持久化数据）：
   ```bash
   docker run -d -p 8501:8501 \
     -v $(pwd)/data:/app/data \
     -v $(pwd)/chroma_db:/app/chroma_db \
     --env-file .env \
     --name rag-lab-ui rag-lab:v1
   ```

## 方案三：Hugging Face Spaces 或 Zeabur 托管（适合对外展示与免维护）
如果你不想购买服务器并维护环境，PaaS 云托管平台是最棒的展示途径。

**部署到 Hugging Face Spaces (Streamlit 应用)**：
1. 在 Hugging Face 注册并新建一个 **Space**，选择 `Streamlit` 作为 SDK。
2. 将 `rag-lab` 的代码上传至该 Space 的仓库。
3. 把依赖写入 `requirements.txt`：
   ```text
   chromadb>=0.5,<0.6
   openai>=1.0.0
   python-dotenv>=1.0.0
   streamlit>=1.38.0
   ```
4. 在 Space 里的 `Settings -> Variables and secrets` 中配置好 `OPENAI_API_KEY` 等环境变量。
5. 平台会自动完成构建并分配给你一个公开的访问链接。

---

## 方案四：全栈私有化部署（结合 Ollama 本地化模型）
由于项目最初设计是学习型 MVP 且希望支持本地运行，因此可以结合本地大模型实现**真正的私有化知识库**，防止数据上传云端。

**部署步骤**：
1. 在性能较好的主机（Mac M系列或带 GPU 的机器）上安装 [Ollama](https://ollama.ai/)。
2. 下载本地分析和 Embedding 模型，例如：
   ```bash
   ollama run qwen2.5:7b       # 用于对话
   ollama pull nomic-embed-text # 用于本地 Embedding
   ```
3. 修改代码中 `openai_provider.py`，将 API Base URL 指向本地的 Ollama OpenAI 兼容接口（默认 `http://localhost:11434/v1`）。
4. `.env` 配置为：
   ```env
   RAG_PROVIDER=openai
   OPENAI_API_KEY=ollama
   OPENAI_BASE_URL=http://localhost:11434/v1
   OPENAI_EMBED_MODEL=nomic-embed-text
   OPENAI_CHAT_MODEL=qwen2.5:7b
   ```
5. 使用**方案一**或**方案二**启动 Streamlit 应用即可使用完全断网的私有化 RAG。
