# 任务池（P0/P1/P2）

## P0（必须）

### AG-001 商户知识模板与校验
- 目标：定义标准模板并拦截缺字段数据。
- 依赖：无。
- 验收：缺“商品信息/门店信息/配送规则/售后政策”时返回缺失项。

### AG-002 问答协议模型
- 目标：统一输出 `answer/sources/risk_flag/handoff`。
- 依赖：AG-001。
- 验收：协议对象可序列化，默认值正确。

### AG-003 风险拦截与转人工
- 目标：敏感问题不自动答，直接 handoff。
- 依赖：AG-002。
- 验收：投诉/退款类问题 `handoff=true`。

### AG-004 编排层（Orchestrator）
- 目标：统一路由正常问答与风险流程。
- 依赖：AG-002, AG-003。
- 验收：单轮链路可跑通且日志可追踪。

### AG-005 API 最小闭环
- 目标：提供 `/health`、`/chat`、`/handoff`。
- 依赖：AG-004。
- 验收：接口响应码和字段契约满足定义。

## P1（增强）

### AG-101 短会话记忆
- 目标：最近 N 轮上下文注入检索问题。
- 验收：追问语义连贯性提升。

### AG-102 成交引导策略
- 目标：非风险场景下输出促单建议。
- 验收：策略命中日志可观测。

### AG-103 指标聚合
- 目标：输出响应时长、转人工率、应答率。
- 验收：指标计算可单测验证。

## P2（后续）

### AG-201 混合检索
- 目标：BM25 + 向量联合召回。

### AG-202 自动评测
- 目标：构建离线评测集，持续评估回答质量。

### AG-203 多渠道接入
- 目标：统一接入平台 IM 与私域消息通道。
