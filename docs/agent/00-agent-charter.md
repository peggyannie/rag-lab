# AI Agent 开发宪章（Claude Code / Codex）

## 1. 目标
- 在 `rag-mvp` 基础上，30 天内交付可试点的“客服成交 Agent”P0 版本。
- 通过小步迭代，让 AI 编程 Agent 能稳定、可验证地完成开发任务。

## 2. 非目标
- 不在本期做全渠道深度打通。
- 不在本期做复杂 CRM/工单系统替代。
- 不做大规模架构重写，优先增量演进。

## 3. 交付定义（Definition of Done）
- 功能：对应任务卡验收标准全部满足。
- 质量：新增测试通过，且关键回归通过。
- 可运维：有基础日志、错误处理、回滚说明。
- 文档：接口/配置/验证命令同步更新。

## 4. 开发原则
- 先测试后实现（TDD 优先）。
- 每次只解决一个明确问题（YAGNI）。
- 小提交、可回滚、提交信息可追踪。
- 任何“完成”声明必须附验证命令和结果。

## 5. 风险红线
- 涉及投诉/退款/赔偿/法律等问题时，不得自动编造答案，必须转人工。
- 对不确定信息必须给出“需人工确认”兜底响应。
- 未跑测试不得声称修复完成。

## 6. 执行节奏
- 日节奏：任务卡 -> 实现 -> 测试 -> 文档 -> 提交。
- 周节奏：周目标 -> 中期回顾 -> 周验收 -> 下周计划。

## 7. 角色约定
- 人类负责人：定义优先级、验收口径、风险策略。
- AI Agent：按任务卡执行开发、测试、文档更新，并输出证据。
